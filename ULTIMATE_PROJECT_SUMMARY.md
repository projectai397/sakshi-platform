## What Was Built

### Core Platform Features

The Sakshi Platform is a full-stack web application built with modern technologies, featuring a unique triple pricing system that allows users to purchase items with money, seva tokens (earned through volunteering and donations), or request them as donations. The platform encompasses multiple interconnected systems including e-commerce, spiritual retreats, caf√© locations, meditation resources, and a volunteer management system.

### Technical Architecture

Built on a robust technology stack including React with TypeScript for the frontend, Node.js with Express for the backend, and PostgreSQL for data persistence, the platform demonstrates enterprise-grade architecture with proper separation of concerns, type safety, and scalable design patterns. The system supports multiple deployment options including Railway, Vercel, DigitalOcean, AWS, and Docker-based deployments.

### Payment Systems

Three complete payment integration systems have been implemented: Razorpay for comprehensive Indian payment methods (cards, UPI, netbanking, wallets), direct UPI integration with QR code generation and mobile deep links, and cryptocurrency support through Coinbase Commerce for Bitcoin, Ethereum, USDC, and other digital currencies. Each system includes webhook handling, payment verification, and refund processing capabilities.

### AI-Powered Features

The platform incorporates advanced AI capabilities through OpenAI's GPT-4, including a 24/7 customer support chatbot with spiritual awareness and platform-specific knowledge, and an intelligent search system that understands natural language queries, automatically extracts filters, and ranks results based on relevance. The AI systems provide personalized recommendations and context-aware responses.

### Design & User Experience

## Quick Start Guide

### For Immediate Deployment

Navigate to the sakshi directory and run the deployment script: `./scripts/deploy-railway.sh`. This will guide you through creating a Railway account, setting up the database, configuring environment variables, and deploying the application. The entire process takes approximately 30 minutes.

### For Local Development

Run the local testing script: `./scripts/test-local.sh`. This will install dependencies, set up the database, seed sample data, and start the development server. Access the platform at http://localhost:3000.

### For Service Configuration

Execute the configuration wizard: `./scripts/configure-services.sh`. This interactive tool will guide you through setting up OAuth, S3 storage, email services, and analytics with your preferred providers.
